# 校园LLM后端开发进度与问题追踪

## 1. 认证模块测试现状
- 认证相关API（注册、登录、用户信息、登出、更新用户）已实现并覆盖详细测试用例。
- 目前19个认证相关测试用例中，14个已通过，5个失败。

## 2. 主要问题与修复进展
### 2.1 数据库邀请码问题
- 之前测试数据库未能正确插入邀请码，导致注册/登录用例失败。
- 已修复：init_db.py 批量插入10条未使用的邀请码，并补充 description 字段，现数据库邀请码充足。
- 建议：兼容老测试码 INVITE2025，避免测试用例因邀请码不符失败。

### 2.2 认证依赖与401/403问题
- FastAPI 默认 HTTPBearer 无token时返回403，部分测试用例要求401。
- 已自定义 HTTPBearer401，部分接口已返回401，但仍有部分接口返回403，需进一步patch。
- 建议：全局替换依赖，确保所有需要token的接口都用自定义security。

### 2.3 用户名唯一性校验
- update_me 接口已修复用户名唯一性校验（排除自己），但部分测试用例仍未通过，需进一步排查。

### 2.4 认证链路
- token生成/解码已修复，sub字段类型一致，token相关接口已基本打通。

## 3. 后续建议
- 彻底修复所有无token场景返回401，保证与API文档和测试用例一致。
- 检查所有测试用例用的邀请码，确保数据库中均有对应数据。
- 持续保持测试驱动开发，修复每一个失败点后及时回归全量测试。
- 认证模块通过后，继续推进课程、文件、聊天等其它模块的实现与测试。
