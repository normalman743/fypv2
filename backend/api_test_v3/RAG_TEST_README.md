# RAG功能测试指南

## 问题说明

在测试过程中发现，RAG功能当前使用的是**硬编码的模拟数据**，这是因为系统检测到没有真实的OpenAI API密钥，自动切换到了Mock模式。

## 系统模式说明

### Mock模式（当前默认）
- **触发条件**: 没有配置有效的OpenAI API密钥
- **RAG行为**: 使用预设的模拟数据源
- **模拟数据源**:
  - 校园设施指南.pdf
  - 数据结构第一讲.pdf  
  - 计算机网络基础.docx
  - 课程大纲.pdf
  - 学习指南.md

### Production模式
- **触发条件**: 配置了有效的OpenAI API密钥且RAG服务可用
- **RAG行为**: 使用真实的向量检索和LLM生成

## 测试RAG功能的步骤

### 步骤1: 准备测试文档
```bash
# 在api_test_v3目录下
python prepare_test_documents.py
```

这个脚本会上传3个测试文档：
- 校园设施指南.txt（包含体育馆、图书馆开放时间等信息）
- 数据结构课程大纲.txt（CS101课程相关信息）
- 学习指南.txt（学习方法和资源指导）

### 步骤2: 运行RAG测试
```bash
python test_chat_message.py
```

测试会提示是否需要上传文档，如果已经执行了步骤1，选择N继续。

### 步骤3: 观察RAG结果

#### Mock模式下的表现：
- RAG sources字段包含随机选择的模拟文件名
- AI回答是基于模板生成的通用内容
- 处理速度快（无需真实LLM调用）

#### Production模式下的表现：
- RAG sources包含实际检索到的文档片段
- AI回答基于真实文档内容生成
- 处理时间较长（需要向量检索+LLM生成）

## 配置Production模式（可选）

如果要测试真实的RAG功能，需要：

1. **配置OpenAI API密钥**:
   ```bash
   export OPENAI_API_KEY="sk-your-real-api-key"
   ```

2. **重启服务**:
   ```bash
   uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
   ```

3. **验证模式**:
   - 检查服务启动日志，应该看到"🚀 Real RAG Service initialized"
   - 如果看到Mock模式相关日志，说明仍在使用模拟模式

## 故障排除

### 1. 超时错误
- **症状**: HTTPConnectionPool Read timed out
- **解决**: 测试已增加超时时间到120秒，如仍超时说明RAG处理异常

### 2. RAG sources为空
- **症状**: rag_sources字段为空数组
- **原因**: 文档未成功处理或向量索引未建立
- **解决**: 重新上传文档，等待处理完成

### 3. 文档上传失败
- **症状**: 文件上传返回错误
- **检查**: 确保课程和文件夹存在，执行reset_system.py重置

## 注意事项

1. **Mock模式是正常的**: 在没有OpenAI API密钥的情况下，系统设计为自动使用Mock模式
2. **文档处理需要时间**: 即使在Production模式下，文档上传后需要时间进行向量化处理
3. **成本考虑**: Production模式会产生OpenAI API调用费用
4. **测试环境**: 当前测试环境主要验证API接口和数据流，Mock模式已足够

## 总结

当前的"硬编码"RAG结果是系统的**预期行为**，不是bug。这是一个智能的降级机制，确保在没有外部API依赖的情况下，系统仍能正常演示和测试核心功能。
