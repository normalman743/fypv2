# 文件权限架构设计文档

## 概述

本文档详细分析校园LLM系统的文件权限需求，并提出可扩展的权限架构设计。

## 当前权限现状

### 存在的权限层次
1. **用户层**: `User.role` (admin, user)
2. **课程层**: `Course.user_id` (课程所有者)
3. **文件层**: `File.user_id` + `File.visibility`
4. **共享层**: `FileShare` 表 (新增)

### 已识别的安全问题
- ✅ 私有文件权限检查正确
- ⚠️ 课程文件权限检查不完整
- ⚠️ 缺少课程成员概念
- ⚠️ 文件共享权限验证待完善

## 权限需求分析

### 1. 课程级权限需求

#### 课程角色定义
```
课程所有者 (Course Owner)
├── 完全控制权：创建/删除课程，管理成员，管理所有文件
│
课程协作者 (Course Collaborator) 
├── 管理权：添加文件，编辑课程文件，查看统计
│
课程成员 (Course Member)
├── 查看权：访问课程文件，下载资料
│
访客 (Guest)
└── 无权限或仅公开文件
```

#### 课程文件可见性规则
- `course` 可见性：课程成员及以上可访问
- `private` 可见性：仅文件所有者可访问
- `public` 可见性：所有人可访问

### 2. 文件级权限需求

#### 文件操作权限矩阵

| 角色 | 查看 | 下载 | 编辑 | 删除 | 共享 |
|------|------|------|------|------|------|
| 文件所有者 | ✅ | ✅ | ✅ | ✅ | ✅ |
| 课程所有者 | ✅ | ✅ | ✅ | ✅ | ✅ |
| 课程协作者 | ✅ | ✅ | ✅ | ❌ | ✅ |
| 课程成员 | ✅ | ✅ | ❌ | ❌ | ❌ |
| 共享用户 | 根据权限级别 | 根据权限级别 | 根据权限级别 | ❌ | 根据设置 |
| 管理员 | ✅ | ✅ | ✅ | ✅ | ✅ |

### 3. 共享权限需求

#### 共享类型
1. **用户共享**: 直接共享给特定用户
2. **课程共享**: 共享给整个课程
3. **组共享**: 共享给用户组
4. **链接共享**: 通过链接分享 (未来功能)

#### 权限级别
- `read`: 查看和下载
- `comment`: 添加评论 (未来功能)
- `edit`: 编辑文件内容
- `manage`: 管理权限，可以再次共享

## 推荐的权限架构

### 方案A: 简化权限模型 (推荐)

#### 核心原则
1. **文件所有者优先**: 文件所有者拥有完全控制权
2. **课程权限继承**: 课程成员自动获得课程文件的基础权限
3. **显式共享覆盖**: FileShare 设置可以覆盖默认权限

#### 实现策略

```python
def check_file_permission(file_id: int, user_id: int, action: str) -> bool:
    """
    权限检查优先级:
    1. 文件所有者 -> 完全权限
    2. 管理员 -> 完全权限 
    3. 显式共享权限 -> 按 FileShare 设置
    4. 课程权限继承 -> 按课程成员身份
    5. 公开文件 -> 读取权限
    6. 拒绝访问
    """
```

#### 数据库设计调整

```sql
-- 新增课程成员表
CREATE TABLE course_members (
    id INT PRIMARY KEY,
    course_id INT NOT NULL,
    user_id INT NOT NULL,
    role VARCHAR(20) DEFAULT 'member', -- 'owner', 'collaborator', 'member'
    joined_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    
    UNIQUE KEY unique_course_member (course_id, user_id),
    FOREIGN KEY (course_id) REFERENCES courses(id),
    FOREIGN KEY (user_id) REFERENCES users(id)
);

-- 更新 FileShare 表支持过期时间
ALTER TABLE file_shares ADD COLUMN expires_at DATETIME NULL;
ALTER TABLE file_shares ADD COLUMN is_active BOOLEAN DEFAULT TRUE;
```

## 示例场景

### 场景1: 多教师协作课程
```
课程: "数据结构与算法"
- 主讲教师 (所有者): 张教授
- 助教 (协作者): 李助教, 王助教  
- 学生 (成员): 100名学生

权限分配:
- 张教授: 所有权限
- 助教: 可以上传/编辑课件，不能删除张教授的文件
- 学生: 可以查看/下载所有课程资料
```

### 场景2: 跨课程文件共享
```
用户: 张教授同时教授"数据结构"和"算法设计"
需求: 将"数据结构"课程的某些文件共享给"算法设计"课程

实现: 使用 FileShare 表，shared_with_type='course'
```

### 场景3: 学生作业提交
```
需求: 学生提交个人作业，只有该学生和课程教师可以访问

实现: 
- scope='course' (属于课程)
- visibility='private' (仅限特定人员)
- 通过课程成员权限控制访问
```
