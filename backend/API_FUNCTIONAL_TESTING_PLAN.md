# 校园LLM系统 API 功能测试计划

## 概述

本文档提供了校园LLM系统所有API功能的全面测试计划。测试覆盖认证、用户管理、学期课程管理、文件管理、聊天消息、AI功能和系统管理等核心模块。

## 测试环境配置

### 基础配置
- **测试服务器**: `https://api-icu.584743.xyz` (本地开发环境)
- **API版本**: `/api/v1/`
- **认证方式**: Bearer Token
- **测试数据库**: 独立的测试数据库实例

### 测试工具
- **HTTP客户端**: Python requests / curl
- **测试框架**: pytest
- **数据验证**: JSON Schema验证
- **性能测试**: 并发请求测试

## 测试策略

### 1. 分层测试策略
- **单元测试**: 单个API端点功能验证
- **集成测试**: API间交互和数据流测试
- **端到端测试**: 完整业务流程测试
- **性能测试**: 响应时间和并发处理能力

### 2. 测试数据管理
- 使用独立的测试数据库
- 每个测试用例使用隔离的测试数据
- 测试完成后自动清理数据

### 3. 认证测试策略
- 无效token测试
- 过期token测试
- 权限不足测试
- 跨用户数据访问测试

## 详细测试计划

## 模块1: 认证与用户管理 API

### 1.1 用户注册测试
**测试端点**: `POST /api/v1/auth/register`

#### 测试用例:
1. **正常注册流程**
   - 输入: 有效邮箱、用户名、密码、邀请码
   - 预期: 成功创建用户，返回用户信息
   - 验证: 用户数据正确保存，密码已加密

2. **邀请码验证**
   - 输入: 无效邀请码
   - 预期: 返回错误，注册失败
   - 验证: 错误信息准确

3. **邮箱格式验证**
   - 输入: 无效邮箱格式
   - 预期: 返回格式错误
   - 验证: 邮箱格式验证正确

4. **用户名重复检查**
   - 输入: 已存在的用户名
   - 预期: 返回用户名已存在错误
   - 验证: 唯一性约束生效

5. **密码强度验证**
   - 输入: 弱密码（少于6位）
   - 预期: 返回密码强度不足错误
   - 验证: 密码策略执行正确

### 1.2 用户登录测试
**测试端点**: `POST /api/v1/auth/login`

#### 测试用例:
1. **正常登录流程**
   - 输入: 正确用户名和密码
   - 预期: 返回访问token和用户信息
   - 验证: token格式正确，用户信息完整

2. **错误凭据测试**
   - 输入: 错误用户名或密码
   - 预期: 返回认证失败错误
   - 验证: 不泄露用户存在信息

3. **账户锁定测试**
   - 输入: 多次错误密码
   - 预期: 账户被临时锁定
   - 验证: 安全机制生效

### 1.3 用户信息管理测试
**测试端点**: `GET /api/v1/auth/me`, `PUT /api/v1/auth/me`

#### 测试用例:
1. **获取用户信息**
   - 输入: 有效token
   - 预期: 返回完整用户信息
   - 验证: 数据准确性和完整性

2. **更新用户信息**
   - 输入: 新的用户名、语言偏好等
   - 预期: 信息更新成功
   - 验证: 数据库记录正确更新

3. **权限验证**
   - 输入: 无效token
   - 预期: 返回401未授权错误
   - 验证: 认证中间件正确工作

## 性能测试计划

### 1. 并发用户测试
- **目标**: 验证系统在高并发下的稳定性
- **测试场景**: 100个并发用户同时使用
- **指标**: 响应时间 < 2秒，错误率 < 1%

### 2. 文件上传性能测试
- **目标**: 验证大文件上传性能
- **测试场景**: 上传100MB文件
- **指标**: 上传速度 > 1MB/s，处理时间合理

### 3. AI响应性能测试
- **目标**: 验证AI回复的响应速度
- **测试场景**: 各种复杂度的查询
- **指标**: 首字节时间 < 1秒，完整响应 < 10秒

### 4. 数据库性能测试
- **目标**: 验证数据库查询性能
- **测试场景**: 大量数据的查询操作
- **指标**: 查询时间 < 500ms，索引效率

3. **并发处理**: 数据库连接池和锁机制
4. **数据一致性**: 事务管理和数据验证
